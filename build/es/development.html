Desarrollo

Estamos interesados en cualquier contribución. Hay muchas maneras de contribuir:

  • Si escribiste algo interesante usando SymPy, por favor envíanos un parche con el código (ver abajo para mayor información).
  • Si encuentras un bug (o quieres decirnos lo que piensas), dínolo en la lista de correos. También puedes añadir un reporte de bug (o comentario) dentro de issues.
  • Crea tu blog acerca de como usas SymPy.
  • Puedes ayudarnos a escribir documentación (simplemente envíanos un parche).
  • Puedes ayudar a las personas en nuestra lista de correo.
  • Puedes ayudar con la revisión de parches.
  • Puedes enviarnos parches con correcciones de errores o nuevas funciones.

La mejor manera de aportar código o documentación es enviarnos un parche.

Cómo enviar un parche

La mejor manera es enviar un pull request en github al repositorio sympy/sympy. Lo revisaremos y haremos push.

Si necesitas ayuda con git, github, pull requests, parches, código, SymPy o algo relacionado, solo envía un correo a la lista de correos o pregunta en Gitter, estaremos felices de ayudar. También, puedes leer nuestras instrucciones cómo preparar parches usando git y github.

Un pull request en github es el método preferido, ya que se nos hace más fácil revisar y hacer push al código. También puedes clonar nuestro repositorio en git (mira el enlace a la derecha), prepara un branch con tu código, súbelo a algún lugar (por ejemplo github) y envíanos el enlace a la lista de correo de sympy, o puedes enviarnos parches brutos --- pero esto implicaría más trabajo para integrarse.

SymPy tiene un amplio equipo de desarrollo, envíanos un parche y tu nombre aparecerá allí también.
