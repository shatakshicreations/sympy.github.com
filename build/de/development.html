Entwicklung

Wir sind an allen Beiträgen interessiert. Es gibt viele Möglichkeiten, wie du beitragen kannst.:

  • Wenn du etwas interessantes mit SymPy geschrieben hast, sende uns bitte einen Patch mit dem Code (siehe unten für weitere Informationen).
  • Wenn du einen Fehler gefunden hast (oder uns gerne mitteilen möchtest, was du denkst), melde dich auf Mailingliste. Du kannst auch einen Fehlerbericht (oder einen Kommentar) in die issues eintragen.
  • Du kannst über die Verwendung von SymPy bloggen.
  • Du kannst uns bei der Dokumentation helfen (sende uns einfach einen Patch).
  • Du kannst Leuten auf unserer Mailingliste helfen.
  • Du kannst helfen, Patches zu evaluieren..
  • Du kannst uns Patches mit Fehlerbehebungen oder neuen Funktionen senden.

Die beste Möglichkeit, Code oder Dokumentation beizutragen, ist es, uns einen Patch zu senden.

Wie sende ich einen Patch

Die beste Möglichkeit ist es, einen Pull Request zu senden gegen das sympy/sympy Repository. Wir werden ihn dann überprüfen und aufnehmen.

Wenn du Hilfe zu git, github, Pull Requests, Patches, Coding, SymPy oder weiterem benötigst, sende uns einfach eine E-Mail an die Mailingliste oder frag uns auf Gitter, wir werden uns freuen, dir zu helfen. Du kannst auch unsere Anleitungen lesen wie man Patches mit git und GitHub erstellt.

Ein Pull Request auf github ist die bevorzugte Möglichkeit, da es einfach für uns ist den Code zu überprüfen und zu übernehmen. Dennoch kannst du auch das aktuelle git Repository (siehe den Link rechts) klonen, einen Branch mit deinen Code erstellen und dort hochladen (z.B. auf github), und einen Link zur Mailingliste senden, oder du kannst uns auch einfach Patches schicken --- aber es bedeutet für uns mehr arbeit diese zu integrieren.

SymPy hat ein großes Entwicklungsteam, sende uns einen Patch und dein Name wird dort auch erscheinen.
